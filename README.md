# 課程實景美感拍照助理 APP｜V0.5 iPhone PWA 測試版

## 一、版本定位

本版本是「課程實景美感拍照助理 APP」的 iPhone PWA 測試版，主要用於測試：

1. iPhone Safari 是否能開啟相機取景。
2. 取景時是否能顯示「美感原則＋一句式拍攝建議」。
3. 若即時相機無法啟動，是否能改用 iPhone 原生拍照／選照片作為備援。
4. 拍照後是否能顯示照片預覽與簡易美感建議卡片。

本版本尚未接入真正 AI 自動判斷，也尚未接 Google Drive／Google Sheet。主要目的為確認 iPhone 現場操作流程與提示介面是否合理。

## 三、重要限制

iPhone 的相機串流功能通常不能在以下環境正常啟動：

1. 直接打開本機 HTML 檔。
2. 在 ChatGPT 檔案預覽中開啟。
3. 使用非 HTTPS 網址。
4. 在某些 App 內建瀏覽器中開啟。
5. 尚未允許 Safari 相機權限。

因此，正式測試請務必將整個資料夾上傳到 HTTPS 網址，例如 GitHub Pages、Netlify、Cloudflare Pages 或其他支援 HTTPS 的網站空間。

---

## 四、iPhone 測試方式

### 方法一：即時相機串流測試

1. 將整個資料夾上傳到 HTTPS 網址。
2. 用 iPhone 的 Safari 開啟該網址。
3. 按下「啟動相機」。
4. Safari 跳出權限詢問時，選擇「允許」。
5. 若成功，畫面會出現即時取景畫面。
6. 取景畫面上方會顯示美感原則提示，例如：
   - `比例｜靠近手部與教具`
   - `韻律｜側前方拍討論動線`
   - `調和｜避開背景雜物`

### 方法二：備援拍照模式

若「啟動相機」失敗，請改用：

1. 按下「備用：使用 iPhone 拍照／選照片」。
2. 選擇「拍照」或「照片圖庫」。
3. 拍完或選完照片後，APP 會顯示照片預覽。
4. 頁面會顯示拍後美感建議卡片。

---

## 五、加入 iPhone 主畫面

1. 用 Safari 開啟已部署的 HTTPS 網址。
2. 點選 Safari 下方的「分享」按鈕。
3. 選擇「加入主畫面」。
4. 名稱可改為「美感拍照助理」。
5. 按「加入」。
6. 之後可從 iPhone 主畫面直接開啟。

注意：加入主畫面後仍需確認相機權限；若相機串流無法使用，仍可用備援拍照模式。

## 八、相機無法啟動時的檢查清單

若 iPhone 無法啟動相機，請依序檢查：

1. 是否用 Safari 開啟，而不是其他 App 內建瀏覽器？
2. 網址是否為 `https://` 開頭？
3. 是否曾拒絕相機權限？
4. 是否在 iPhone 設定中允許 Safari 使用相機？
5. 是否直接開啟本機檔案？若是，請改上傳 HTTPS 後測試。
6. 若仍失敗，請使用「備用：使用 iPhone 拍照／選照片」。

---

## 九、建議測試重點

現場測試時，請特別觀察：

1. 提示文字是否太長。
2. 提示是否遮住取景畫面。
3. 美感原則名稱是否看得懂。
4. 「比例、均衡、調和、韻律」等提示是否真的能幫助拍照。
5. 備援拍照是否比即時相機更穩定。
6. 在課程現場是否來得及看提示並調整位置。
